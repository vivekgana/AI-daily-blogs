### 3. Artifacts (Downloaded Logs)

When workflows fail or complete, logs are uploaded as **artifacts**.

**How to Access Artifacts:**

#### Via Web Interface:
1. Go to: https://github.com/vivekgana/AI-daily-blogs/actions
2. Click on a workflow run
3. Scroll to bottom of the page
4. See **"Artifacts"** section
5. Click to download ZIP file

**Available Artifacts:**

| Artifact Name | Contains | When Created | Retention |
|--------------|----------|--------------|-----------|
| `error-logs` | Application logs from `logs/` directory | On failure | 7 days |
| `unit-test-results` | HTML test report + coverage | Always | 30 days |
| `integration-test-results` | HTML test report | Always | 30 days |
| `test-results-{suite}` | Test reports from manual runs | Always | 30 days |

## ðŸ”§ Workflow Log Configuration

### Current Configuration:

#### 1. **generate-daily-blog.yml**
```yaml
- name: Upload logs on failure
  if: steps.generate.outputs.status != 'success'
  uses: actions/upload-artifact@v4
  with:
    name: error-logs
    path: logs/              # â† Application logs directory
    retention-days: 7
```

#### 2. **run-tests.yml**
```yaml
- name: Upload unit test results
  if: always()
  uses: actions/upload-artifact@v3
  with:
    name: unit-test-results
    path: |
      unit-test-report.html
      htmlcov/
```

#### 3. **test-on-demand.yml**
```yaml
- name: Upload test results
  if: always()
  uses: actions/upload-artifact@v3
  with:
    name: test-results-${{ github.event.inputs.test_suite }}
    path: |
      test-report.html
      htmlcov/
```

## ðŸ” Searching Logs

### In Web Interface:

1. **Open workflow run logs**
2. **Click search icon (ðŸ”)** in top right
3. **Type search term:**
   - `ERROR` - Find all errors
   - `FAILED` - Find failed tests
   - `404` - Find specific error codes
   - `test_kaggle` - Find specific test
4. **Navigate results** with up/down arrows

### Common Search Terms:

| Search For | To Find |
|-----------|---------|
| `ERROR` | All error messages |
| `FAILED` | Failed test cases |
| `Traceback` | Python stack traces |
| `401` or `404` | HTTP error codes |
| `AssertionError` | Test assertion failures |
| `PASS` | Successful operations |
| `Skipped` | Skipped tests |

## ðŸ› Debugging with Logs

### Example: Finding Why a Test Failed

1. **Go to failed workflow run**
2. **Click on failed job** (red X)
3. **Click on failed step**
4. **Search for "FAILED"**
5. **Look for error message above "FAILED"**

**Example Error Log:**
```
tests/integration/test_collectors_integration.py::TestKaggleCollectorIntegration::test_get_competitions

E   AssertionError: assert 0 > 0
E    +  where 0 = len([])

tests/integration/test_collectors_integration.py:45: AssertionError

Short test summary info:
FAILED tests/integration/test_collectors_integration.py::TestKaggleCollectorIntegration::test_get_competitions - AssertionError: assert 0 > 0
```

This tells you:
- **File:** `test_collectors_integration.py`
- **Test:** `test_get_competitions`
- **Line:** 45
- **Issue:** No competitions returned (empty list)

## ðŸ’¡ Pro Tips

### 1. Enable Debug Logging

Add to workflow file:
```yaml
env:
  ACTIONS_RUNNER_DEBUG: true
  ACTIONS_STEP_DEBUG: true
```

Shows more detailed GitHub Actions internal logs.

### 2. Add Custom Log Statements

In your Python code:
```python
import logging
logger = logging.getLogger(__name__)

logger.info("Starting blog generation...")
logger.debug(f"API response: {response}")
logger.error(f"Failed to fetch data: {error}")
```

### 3. Use Workflow Summaries

In workflows, add:
```yaml
- name: Create summary
  run: |
    echo "## Test Results" >> $GITHUB_STEP_SUMMARY
    echo "âœ… 25/25 tests passed" >> $GITHUB_STEP_SUMMARY
```

Shows summary on workflow run page.

### 4. Set Up Alerts

- Configure email notifications (already done in your workflows)
- Create GitHub issues on failure (already configured)
- Use Slack/Discord webhooks (optional)
